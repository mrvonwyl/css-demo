General Info

Units

px absolute 1 pixel on your screen
% relative relative to the size of the parent element
em relative relative to the font size of the element
rem relative relative to the font size of the root (html) element
vw relative relative to 1% of the viewport width (100vw is the entire width of the viewport)
vh relative relative to 1% of the viewport height (100vh is the entire height of the viewport)

Selectors

selectors are a matching pattern to apply the styles to e certain set of elements

Selector Description Example
* all elements *
[htmlTag] all [htmlTag] elements div
#[id] all elements that have a matching id attribute #content
.[class] all elements with a class attribute that contains [class] button
[selectorA] [selectorB] all elements that match [selectorB] that have a element that matches [selectorA] anywhere in their parent tree #content .spacer
[selectorA] > [selectorB] all elements that match [selectorB] that have a direct parent that matches [selectorA] #content .spacer
[selectorA] + [selectorB] all elements that match [selectorB] that is placed immediately after an element that matches [selectorA] .button + .button
[selectorA] ~ [selectorB] all elements that match [selectorB] that are placed immediately after an element that matches [selectorA] .button + .button
[selectorA][selectorB] all elements that match [selectorA] and are preceeded by en element that matches [selectorB] (anywhere in the DOM) div.blue
[selectorA], [selectorB] all elements that match either selector h1, h2
[attribute="[value]"] all elements that have an attribute that matches [value]
= > exact match
~= > exact match of [value] for at least one element in a space separated list
|= > starts with whole word [value] or followed by hyphen [value]-*
^= > starts with [value]
$= > ends with [value]
*= > contains [value] 		a[target="_blank"]
:[pseudoClass] special pseudo classes (:hover, :not(), :nth-child()) :hover, :not(:last-child)
:[pseudoElement] special pseudo elements (::before, ::first-line)

Order and priority

Earlier rules are overridden by later rules, if they have the same priority. (no matter if spread over multiple files, the last file is the most relevant)

Priorities (highest first):

  • !important
  • style attribute
  • id selector
  • class selector, pseudo-classes
  • type selector, pseudo-elements
  • universal selectors (*, ~, >, +)

Do's & don'ts

Disclaimer: These are my personal opinions.

  • use F12! it's your best friend
  • use some sort of css reset (or do it your own) do get a decent base and to avoid surpises
  • try to use rem, px for absolute measurements. you might use cm/mm for print css
  • if you have a css problem, try to figure out what property of what element causes it. always remove or exchange properties first, instead of trying to override the with other ones. know what you are doing and know the sideeffects
  • use float as rarly as possible, and if only for floating within a block, not for layouting purposes
  • use tables only for content structuring, not for the page layout
  • display content dependent images with an img tag, not via background property
  • never use inline css, except if it is a programmatical value and you cannot apply additional css classes
